# 行业垂直设计：执行手册

## 一句话总结
**每个垂直行业的设计不是"换个 logo"，是"换一套底层逻辑"**。医疗要 HIPAA 合规 + 临床工作流；金融要 PCI-DSS + 风险披露；工业要 HMI + 操作员控制台；车载要驾驶员注意力管理。**通用 UX 原则适用 70%，剩下 30% 是行业专属**。

## 1. 4 大行业速览

| 行业 | 用户角色 | 核心痛点 | 合规要求 | 设计差异 |
|---|---|---|---|---|
| **医疗** | 医生 / 护士 / 患者 / 行政 | 时间压力 + 病历复杂 + 多系统 | HIPAA / FDA / HL7 / FHIR | 临床工作流 + Epic / Cerner 集成 |
| **金融** | 投资者 / 顾问 / 合规官 / 普通用户 | 风险 + 信任 + 监管 | PCI-DSS / SOX / KYC / AML | 风险披露 + 多角色权限 + 审计 |
| **工业** | 操作员 / 工程师 / 主管 | 安全 + 实时 + 高密度 | IEC 62443 / OSHA / NERC | HMI + SCADA + 操作员控制台 |
| **车载** | 驾驶员 / 乘客 / 后排 | 注意力分散 + 安全 | NHTSA / Euro NCAP / ISO 26262 | 驾驶员分心 + CarPlay / Android Auto |

**共性 30%**：
- 高密度信息
- 多角色权限
- 键盘党友好
- 长会话（8h+）
- 数据可视化是核心
- a11y 必做（不仅合规，还服务真实用户）

**差异 70%**：
- **医疗**：救命为第一原则（错误 = 死亡）
- **金融**：信任为第一原则（错误 = 罚款 + 起诉）
- **工业**：安全为第一原则（错误 = 设备损坏 + 人员伤亡）
- **车载**：注意力为第一原则（错误 = 车祸）

## 2. 医疗行业设计

### 5 大设计原则
1. **救命 > 效率 > 体验**——> 错过一条过敏警告 = 死亡
2. **永远不阻塞主流程**——> 急诊科不能等系统加载
3. **永远可审计**——> 谁在何时做了什么必须可追溯
4. **多角色协作**——> 医生 / 护士 / 药师 / 检验师 / 患者共享病历
5. **离线工作**——> 医院网络常断

### HIPAA 合规设计 6 必做
1. **PHI 加密传输 + 存储**（Protected Health Information）
2. **自动登出**（idle 15-30 分钟）
3. **审计日志**——> 谁看了哪条病历
4. **最小权限**——> 护士看不到精神科病历
5. **截图禁用 / 水印**——> 防止数据外泄
6. **BAA 协议**（Business Associate Agreement）——> 与供应商签

### Epic / Cerner 集成
- **Epic**（美国市占 36%）/ **Cerner**（Oracle 收购）——> 医院 EHR 系统
- **FHIR API**——> 现代集成标准
- **SMART on FHIR**——> 应用内嵌入
- **设计要点**：
  - 永不重复发明轮子（沿用 EHR 视觉语言）
  - 病历数据用 FHIR 资源（Patient / Observation / MedicationRequest）
  - **永远显示数据来源**（避免信任错误数据）

### 5 个真实医疗产品
1. **Epic Hyperspace**——> 医生工作站（Windows 客户端）
2. **Cerner PowerChart**——> 同上
3. **Doximity**——> 医生社交（iOS/Android）
4. **Doximity Dialer**——> 医生专属电话（HIPAA 合规）
5. **Zocdoc**——> 患者预约（移动优先）

### 5 大反模式
1. **用 emoji 当状态**（🏥 = 住院）——> 临床医生没时间理解
2. **淡色背景**——> 手术室强光下看不清，必须高对比
3. **小字号**——> < 14pt 必崩（医生老花眼）
4. **左对齐 + 暗模式**——> 临床用高对比浅色
5. **无限滚动**——> 病历必须可搜索 + 可分页

## 3. 金融行业设计

### 5 大设计原则
1. **信任 > 美观**——> 银行网站不一定美，但必须稳
2. **风险显式披露**——> "投资有风险"不能藏在 8pt 灰字
3. **多角色权限**——> 顾问 / 合规官 / 客户看到不同
4. **永远可逆 + 永远审计**——> 撤销 + 撤销记录
5. **数字必须真实**——> 0.01% 不准 = 监管罚款

### PCI-DSS 合规设计 5 必做
1. **永不存储 CVV**（信用卡背 3 位）——> 罚款 $500k+
2. **PAN 部分遮罩**（`**** **** **** 1234`）——> 日志 / 截图
3. **TLS 1.2+ 强制**——> HTTP = 立刻失败
4. **多因素认证**——> 大额转账必须 MFA
5. **3D Secure**——> 跨境交易欧洲强制

### KYC / AML 设计
- **KYC**（Know Your Customer）——> 实名认证
- **AML**（Anti-Money Laundering）——> 反洗钱
- **设计要点**：
  - OCR 自动识别身份证 / 护照
  - 人脸识别活体检测
  - 风险评分（低 / 中 / 高 风险不同流程）
  - **永远显示"为什么问这个"**——> 合规要求

### 5 个真实金融产品
1. **Stripe Dashboard**——> 商家收款（最现代金融 UI）
2. **Robinhood**——> 散户投资（移动优先）
3. **Wealthfront / Betterment**——> 智能投顾
4. **Bloomberg Terminal**——> 专业投资（PC 客户端，键盘党）
5. **Goldman Sachs Marquee**——> 投行（极端高密度）

### 5 大反模式
1. **3D 渐变 + 紫粉**——> 金融 = 信任 = 克制
2. **隐藏风险**——> "本产品过去 12 个月涨 30%"（不写基准）
3. **模糊数字**——> "$1k+ profit" 而非 "$1,247.32 profit"
4. **无限滑点**——> 数字必须可点击看明细
5. **移动端交易**——> 大额必须二次确认（合规）

## 4. 工业行业设计

### 5 大设计原则
1. **安全 > 效率**——> 错按一下 = 设备爆炸
2. **实时 + 离线备份**——> 控制系统不能崩
3. **高密度信息**——> 操作员看 30+ 仪表
4. **物理环境考虑**——> 戴手套 / 戴眼镜 / 强光 / 噪音
5. **可审计**——> 操作员 A 在 14:32 关闭了安全阀

### HMI 设计 8 必做
1. **大按钮**——> 戴手套也能点（最小 60×60px）
2. **高对比**——> 强光下也清晰（4.5:1+）
3. **色彩编码一致**——> 红=危险 / 黄=警告 / 绿=正常 / 蓝=信息
4. **物理反馈**——> 按下要"咔嗒"声 + 触感
5. **永远显示紧急停止**——> 大红按钮，独立电源
6. **状态显式**——> 仪表盘数字 + 视觉 + 文本三重
7. **可恢复**——> 误操作必须可撤销
8. **离线 fallback**——> 网络断不崩（本地缓存）

### SCADA 监控设计
- **SCADA**（Supervisory Control and Data Acquisition）——> 工业控制系统
- **设计要点**：
  - 流程图优先于表格
  - 颜色 = 状态（红/黄/绿/蓝/灰）
  - 趋势图（sparkline + 历史曲线）
  - 警报分级（critical / warning / info）
  - **告警风暴防护**——> 重复告警折叠

### 5 个真实工业产品
1. **Siemens SIMATIC WinCC**——> 工业 HMI
2. **Rockwell FactoryTalk**——> 同上
3. **Schneider Electric EcoStruxure**——> 能源管理
4. **ABB Ability**——> 工业 IoT
5. **GE Digital Predix**——> 工业大数据

### 5 大反模式
1. **扁平化 + 阴影**——> 工业要写实感（3D 立体）
2. **小字号**——> 操作员眼镜 + 戴口罩 + 噪音环境
3. **隐藏控制**——> 工业永远显式控制
4. **滑动操作**——> 戴手套失败率高，必须点按
5. **动画装饰**——> 工业动画必须有信息（数据流动 / 状态变化）

## 5. 车载行业设计

### 5 大设计原则
1. **驾驶员注意力第一**——> 视线离开路 2 秒 = 55 米盲驾
2. **任务时间 < 12 秒**——> NHTSA 建议单次操作
3. **静态视觉**（驾驶员不操作时）——> 动态视觉分散注意力
4. **物理按钮 + 触屏混合**——> 音量 / 温度必须物理
5. **不阻塞关键信息**——> 速度 / 导航 / 警告永远优先

### 驾驶员分心 4 测
- **NHTSA 眼动偏离测试**——> 视线离开路时间 < 2s
- **Euro NCAP**——> 欧洲新车评分
- **ISO 26262**——> 汽车功能安全标准
- **设计验证**：实验室测 30+ 受试者

### 触屏 vs 物理按钮
- **永远用物理**：音量 / 温度 / 雨刷 / 危险警告灯
- **永远用触屏**：导航目的地 / 媒体选择 / 联系人
- **混合**：空调风量（拨杆）+ 模式（触屏）

### 5 个真实车载系统
1. **Apple CarPlay**——> iOS 投影（2024 年 90%+ 新车支持）
2. **Android Auto**——> Android 投影
3. **Tesla 自研**——> 全触屏 + OTA
4. **Mercedes MBUX**——> 触屏 + 语音 + AR HUD
5. **BMW iDrive**——> 旋钮 + 触屏混合（驾驶友好）

### 5 大反模式
1. **复杂菜单**——> 驾驶员点 3 次菜单 = 5 秒 = 137 米盲驾
2. **滑动操作**——> 必须点按
3. **小字号**——> 行驶中看不清（< 16pt 必崩）
4. **动画 / 视频**——> 完全禁止（除 HUD 速度外）
5. **登录 / 验证**——> 永远不要求驾驶员输入密码

## 6. 4 大行业通用 B2B 设计模式

### 模式 1: 高密度信息
- 14-16px 主体（不放大）
- 4-8px 行高
- 极小 padding（4-8px）
- 紧凑列表（30-36px 行高）
- **参考**：Bloomberg Terminal / Linear / Salesforce

### 模式 2: 多角色权限
- 角色 / 权限 / 工作流是核心
- 界面按角色定制（不是 1 套 UI 给所有人）
- 权限显式（不要"魔法"）
- **参考**：Stripe Connect / Salesforce / Epic Hyperspace

### 模式 3: 数据真实性
- 数字必须可点击看明细
- 时间戳必显示（UTC + 本地时间）
- 数据源 / 置信区间必显示
- **参考**：Bloomberg / Financial Times / Bloomberg Terminal

### 模式 4: 键盘党友好
- 90% 操作键盘完成
- 快捷键显式（? 显示快捷键面板）
- 命令面板（Cmd+K）
- **参考**：Linear / Vercel / Bloomberg Terminal / Superhuman

### 模式 5: 长会话友好
- 8+ 小时工作不疲劳
- 暗色模式默认（OLED）
- 不会 1 小时未保存就丢失
- 撤销栈足够深（最近 50 步）
- **参考**：Figma / Linear / VS Code

## 7. 4 大行业合规速查

| 行业 | 主要合规 | 罚款上限 | 设计影响 |
|---|---|---|---|
| **医疗** | HIPAA / FDA 21 CFR Part 11 / HL7 | $1.5M / 年 / 类型 | PHI 加密 + 审计 + BAA |
| **金融** | PCI-DSS / SOX / KYC / AML / GDPR | $500k+ / 事件 | PAN 遮罩 + MFA + 3DS |
| **工业** | IEC 62443 / OSHA / NERC | 设备损失 + 死亡赔偿 | 紧急停止 + 离线 + 审计 |
| **车载** | NHTSA / Euro NCAP / ISO 26262 | 召回成本 | 驾驶员分心测 + 物理按钮 |

## 8. 4 大行业 a11y 要求

| 行业 | 关键 a11y | 标准 |
|---|---|---|
| **医疗** | 屏幕阅读器（盲人医生）/ 语音输入（手忙） | WCAG 2.2 AA + Section 508 |
| **金融** | 视觉障碍用户（占 5%）/ 老年用户 | WCAG 2.2 AA + ADA |
| **工业** | 戴手套 / 戴眼镜 / 噪音环境 | 物理交互 + 高对比 |
| **车载** | 老年驾驶员 / 视障用户 | WCAG 2.2 AA + ISO 9241 |

## 9. 5 大真实案例对比

### 案例 1: 表单
- **医疗**（Epic）：120 字段 1 屏，按 Tab 跳
- **金融**（Stripe）：6 字段 1 屏，每字段验证
- **工业**（Siemens）：4 字段大按钮，物理确认
- **车载**（CarPlay）：3 字段横滑，避免分心

### 案例 2: 数据展示
- **医疗**：病历时间轴 + 检验结果表格
- **金融**：K 线 + 财报 + 风险指标
- **工业**：流程图 + 仪表盘 + 趋势曲线
- **车载**：HUD 速度 + 导航箭头 + 媒体卡片

### 案例 3: 错误处理
- **医疗**：过敏警告不可关闭 + 必须确认 + 必签名
- **金融**：风控拦截 + 二次验证 + 人工审核
- **工业**：警报声 + 灯光 + 仪表闪烁 + 自动停机
- **车载**：HUD 警告 + 座椅震动 + 方向盘振动

## 10. 5 大常见 B2B 反模式

1. **用 C 端思路做 B2B**——> 消费者应用"克制"，B2B 反而"高密度"
2. **隐藏控制**——> B2B 永远显式控制
3. **简化数据**——> B2B 用户要 100% 真实数据
4. **无障碍是合规事**——> 真实 B2B 用户有 25% 需要 a11y
5. **用 C 端成功指标**——> B2B 用 NPS + 任务完成时间 + 错误率

## 11. 5 大决策问题

### Q1: "我做的是 B2B 还是 B2C？"
- B2B → 高密度 + 键盘党 + 多角色 + 真实数据
- B2C → 简洁 + 引导 + 单一角色 + 启发性数据

### Q2: "用户每天用 8h 还是偶尔用？"
- 8h+ → 暗色默认 + 极小 padding + 命令面板
- 偶尔 → 浅色默认 + 引导教程 + 简单流程

### Q3: "错误成本是什么？"
- 死亡 / 设备爆炸 → 显式确认 + 二次验证
- 经济损失 → 风险披露 + MFA
- 体验差 → undo + 撤销

### Q4: "合规边界在哪？"
- 医疗 / 金融 / 工业 / 车载各有合规框架
- 设计必须从合规倒推

### Q5: "我的用户是专家还是新手？"
- 专家 → 极简 + 快捷键 + 高级设置
- 新手 → 引导 + 帮助 + 安全默认值

## 12. 5 条"如果你在做行业垂直设计，记住这些"

1. **通用 UX 70%，行业专属 30%**——> 30% 决定成败
2. **救命 > 信任 > 体验**——> 医疗救命 / 金融信任 / 工业安全
3. **永远可审计**——> 4 大行业都必须
4. **键盘党优先**——> 4 大行业都是长会话用户
5. **合规倒推设计**——> 不是设计决定合规，是合规决定设计

## 5 条反面教材

1. **用 C 端设计做 Epic**——> Epic 必须高密度，不能 Material You
2. **隐藏金融风险**——> 美国 SEC 罚款 + 集体诉讼
3. **工业用扁平化**——> 戴手套 + 强光下看不见
4. **车载用复杂菜单**——> 驾驶员分心 = 车祸
5. **医疗用 emoji**——> 临床医生没时间理解

## 5 条"如果你只能记一条"

1. **4 大行业共性 30%、差异 70%**——> 70% 决定成败
2. **救命 / 信任 / 安全 / 注意力**——> 4 大行业第一原则
3. **永远可审计 + 永远可逆**——> 4 大行业共同底线
4. **键盘党 + 高密度 + 多角色**——> B2B 三大铁律
5. **合规倒推设计**——> 不是设计决定合规
