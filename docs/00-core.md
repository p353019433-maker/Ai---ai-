# 00 · 核心判断 — 坏 AI 设计是失忆的设计

> 蒸馏自：LESS-AMNESIAC-DESIGN、APPLIED-DESIGN-JUDGMENT、DESIGN-CONFLICTS、DESIGN-ANTI-PORTFOLIO、design-decision-handbook、design-ultimate-handbook、design-philosophy-master（原文见 git 历史，证据见 `../research/batches/`）

## 一、核心命题

> **Bad AI design is not design without beauty. It is design without memory.**

AI 默认设计常被它的可见症状描述：紫蓝渐变、居中 SaaS hero、玻璃卡片、软阴影、sparkles、说着友好废话的文案。这些是症状，不是病。一个设计可以避开所有套路——黑白、editorial、Swiss、brutalist——仍然是 slop，错在同一个地方：**失忆**。

设计忘了形式为何存在，忘了它模仿的东西产生于什么条件。它记得设计成品的平均外观，不记得让那些成品成为必需的压力。所以"换个风格 prompt"（make it Bauhaus / Aesop / brutalist）只是给失忆换戏服。

**设计记忆**不是怀旧，不是加做旧纹理。它意味着制品携带对自身条件的交代，分七层：

| 记忆层 | 它记得什么 | 审问句 |
|---|---|---|
| 媒介 | 屏幕光不是油墨；拇指/光标/注视是不同身体；性能与辅助技术是设计材料 | 这个设计对它生存的媒介知道什么？ |
| 机构 | 剧院不该说 fintech 的话；交通系统要几十年一致性而非 campaign 感 | 谁在说话，它主张什么权威？ |
| 生产 | 字体授权、token、组件、QA、维护决定美能否存活 | 谁来维护，怎么维护？ |
| 阅读 | 行高、行长、光学尺寸、文字方向改变理解；居中先是仪式性的才是可读的 | 真人如何随时间解析它？ |
| 使用 | loading/empty/error/undo/focus 不是边缘情况，是产品随时间的样子 | 之前、之中、之后、失败时发生什么？ |
| 劳动 | 设计系统需要治理；AI 产出借用了人的工作，不能用新奇掩盖抽取 | 谁的工作、负担、不可见性在这个设计里？ |
| 历史 | Swiss 网格来自公共清晰与生产纪律，不是"干净"的情绪；Memphis 是批评不是随机俏皮 | 援引哪个传统？真正继承了它的什么机制？ |

**为什么 AI 会忘**：截图里没有 IA 为何这样选、研究推翻了什么、哪个授权被拒、错误态如何行为、什么因测试失败被删。图像记得最终表面，过程记得理由——AI 继承了表面，没继承理由。这就是 AI 设计常显得过度自信的原因：它有成品的语气，没有决策的伤疤。

**记忆不是更多细节**。加纹理、加微交互、加"人味"瑕疵解决不了失忆——失忆靠"关系"解决：网格关联信息而非只关联对齐；颜色关联感知、状态与文化而非只关联情绪；动效关联状态变化而非只关联取悦；声音关联权力与后果而非只关联友好。稀疏的设计可以深刻地像人，繁复的设计可以只记得别的图像。

## 二、十条可迁移原则

1. **结构先于表面**——复杂问题从 taxonomy、IA、状态、任务流开始；失败气味：漂亮卡片藏着不清不楚的分类。
2. **风格必须继承理由**——Swiss/Memphis/brutalism/flat 只在特定理由下成立；照抄外观不抄理由 = 戏服设计。
3. **字体是基础设施**——语言覆盖、x-height、光学尺寸、授权、生产生态；失败气味：一个时髦无衬线包打所有字号语言情绪。
4. **身份是规则不是锁死的外观**——强身份定义生成性 DNA：字体行为、色彩逻辑、动效语法、变化上限；审问：什么可以变而身份仍可辨认？
5. **交互有时间形态**——静态 mockup 漏掉 loading、反馈、错误、空态、撤销；失败气味：截图好看，没人设计点击之后。
6. **诚实压过取悦**——欢快语气比冷漠更能掩盖不对称；审问：用户此刻以为什么是真的，这个信念配得上吗？
7. **生产决定美能否存活**——设计必须扛住实现、组件化、授权、性能、维护、交接；失败气味：一次性 art direction，没有 token、状态、归属。
8. **语境包含权力**——医疗、可访问性、劳动、公共标识表明设计是社会基础设施；审问：谁获得可见性、负担、控制或排斥？
9. **策展是设计记忆**——没有索引、档案、批评、谱系，设计师重复旧错误，AI 模仿无来历的截图；失败气味：没有出处的 moodboard。
10. **出处纪律是品味的一部分**——设计主张应携带证据等级（deep read / reference / missing / inference）。**幻觉出处产生幻觉品味；没有出处的品味是带脚注的氛围。**

## 三、七组原则冲突与元规则

从不与其他原则碰撞的原则通常只是口号。真判断始于两个都对的原则要求相反的行动。

1. **结构先行 vs 编辑声音**——问用户的第一问题是 **orientation 还是 recognition**：完成任务（支付/搜索/比较）→ 结构赢；感受机构立场（剧院/运动/杂志）→ 声音可领跑，但最终须固化为规则（typographic DNA、节奏、变化上限、可访问性边界）。Public Theater 的喧闹字体就是结构的一部分，因为机构的公共性就是讯息。坏折中：SaaS dashboard 到处用海报字体。好折中：安静任务 UI + 表现型 campaign/空态表面 + 共享识别系统。
2. **字体基础设施 vs 风格理由**——拆成 **workhorse 和 voice**：workhorse 管标签、表单、数字、错误、本地化，要无聊到隐形；voice 管标题、onboarding、编辑时刻。除非家族有足够 breadth 和 optical range，不要一个字体身兼两职。坏折中：漂亮 display serif 用于密集设置标签。
3. **诚实 vs 美**——**美在"增加与现实的接触"时被允许，在"减少与现实的接触"时可疑**。好的数据可视化让气候风险可把握、平静的表单让政府流程少羞辱 = 合法的美；掩盖原因的可爱报错、掩盖未知时长的装饰 spinner = 不诚实。Sagmeister 的提醒同时是警告：魅力可以压倒证据。
4. **出处纪律 vs 交付速度**——区分 **决策置信度与可逆性**：高影响难逆转（品牌名、字体授权、导航模型、健康/金融声明）→ 减速；低影响可逆（图标变体、文案草稿）→ 带显式不确定性交付。诚实的快是"因证据 A/B 临时选这个方向，锁定前测 X"，不是假装知道。
5. **少即是多 vs 状态完备**——**极简只有在所有状态被设计后才有效**。从单帧里删东西是过早的；工作从"装饰削减"转向"行为压缩"：只展示当下重要的，但让下一状态明显且可恢复。静态极简屏显得优雅恰恰因为它藏起了脏活。
6. **平台约定 vs 独特身份**——**绝不自定义用户赖以安全、定向、可访问的部分**。身份住在平台信任层之上：type voice、imagery、动效签名、空态、campaign 表面。交互风险越低，识别越可以发声。坏折中：自定义 dropdown 破坏键盘与读屏。
7. **公共清晰 vs 文化特异**——问 **谁被要求理解，谁被要求感到被代表**：机场标识、急救表格 → 公共清晰；文化节、社区运动 → 可携带内部记忆的形式，但特异性仍欠用户一条可用路径。好折中：导航层用公共语法 + 表达层携带地方记忆。

**元规则**：原则冲突时不要取平均，要命名**信任契约**——①用户在保护什么（时间/金钱/安全/尊严/定向/身份）？②机构需要保护什么（可信度/独特性/包容/法律清晰）？③哪部分可逆？④什么证据会改变这个决定？

> **好设计师不是懂更多原则的人，而是知道今天哪条原则该输的人。**

## 四、十个被修正的认知错觉（反作品集）

格式：旧直觉 → 新规则。

1. "AI 设计是视觉缺陷" → **审 AI 产出先问"这个形式记得什么人类处境？"答案为空 = 失忆**，即使避开了所有已知套路。
2. "干净极简大概率更好" → **没看过 loading/empty/error/partial/disabled/undo/focus/mobile 之前，绝不夸一个极简 UI**。单帧干净画面是试装，不是产品。
3. "字体选择 = 品味+可读性" → 字体选择必须回答两问：**它能扛住什么工作？它携带什么记忆？**两个都答不出 = 纯装饰。
4. "modernist 是一个干净教义" → 是数个互不兼容的现代主义（governance、grammar、signage、public clarity、production constraint…）。**说不出是哪种现代主义就不要说 modernist。**
5. "结构先行近乎普适" → 第一用户问题是 recognition/stance/belonging 时，文化声音可以合法领跑——但仍须变成规则系统而非情绪。
6. "诚实文案是文案问题" → 是权力问题。**欢快语气比冷漠更能掩盖不对称**。改语气前先识别用户被要求交出什么（时间/数据/金钱/确定性/控制/尊严），把这笔交易写到可见。
7. "human-centered = 足够的用户研究" → 系统边界错了照样失败。**研究必须包含用户周围的系统**：设计让个人屏幕更好，却把负担转给护士、审核员、翻译、家属——问题没被解决。
8. "provenance 是研究卫生" → 是审美卫生。假装读过一个来源不只污染参考书目，还污染建立其上的判断。
9. "美 vs 功能是陈旧二元" → 依然成立，但**只有能命名美改变了什么功能**（理解、记忆、参与意愿、信任、节奏、被感知的尊严）才许说"美是功能性的"，说不出就闭嘴。
10. "产出应该是 handbook" → 完美 checklist 防得住明显 slop，产生不了矛盾之下的判断。**每份新综合至少含其一：一个冲突、一个被改变的信念、一次失败的应用、或一个心爱原则该输的案例。**

## 五、三句收束

1. 好设计师知道今天哪条原则该输。
2. 没有出处的品味是带脚注的氛围；幻觉出处产生幻觉品味。
3. 好的设计 90% 是删除，10% 是摆放；AI 缺的是"删掉这一笔"的勇气。

## 六、为什么"感觉不对"：fluency vs intentionality（增量并入）

> 蒸馏自 batch-038/039 + research-notes-05（感知心理学）

- **核心论断：AI 设计"感觉不对" = 高 fluency + 低 intentionality 的冲突**。全局抛光、局部不连贯——"先像 design，才像 a design for something"。
- **能力的恐怖谷（uncanny valley of competence）**：业余作品可见地不完美，专家作品可见地有意图；AI 居中——专家级抛光 + 模板级推理。
- **意图密度（intention density）**：每个决策（颜色/radius/间距/节奏）能否被解释。AI 高 fluency 分、低 intention 分——过缩略图，但注意力一停留就露馅。
- **diagnostic detail**：不可被同义替换且不损失意义的细节。AI 满是 non-diagnostic detail（glow/card/gradient 可随意互换）。**人类信任始于 edge case，AI 回避 edge case 因为它破坏 generic beauty。**
- **statistical average vs specific choice**：具体选择带 opportunity cost（选粗糙黑白照=拒绝抛光渐变）；AI 回避可见的牺牲感，这种"想被所有人喜欢"可被感知。

**六条 anti-AI 原则（带 audit 问句）**：
1. 每个感知线索必须编码意义。
2. situated 调色板——问"这调色板能属于任何 AI 创业公司吗？"
3. 用不对称揭示优先级——问"系统在哪假装事物平等而它们其实不平等？"
4. 保留有用的 grain——settings 朴素 / campaign 表达 / error 直白。
5. 定义 variation 语法——问"用户看三次能学会它的含义吗？"
6. 区分 polish（表面一致）vs care（对情境的适当性）。
