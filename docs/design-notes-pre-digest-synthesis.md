## 1. Müller-Brockmann + Vignelli（Wikipedia 真实资料）

**Müller-Brockmann (1914-1996)**
- Swiss graphic designer, "pioneer of International Typographic Style"
- 教授 1957-1960 苏黎世艺术大学
- **客户：IBM 欧洲设计顾问 (1967)**
- **最爱字体：Akzidenz-Grotesk**（不是 Helvetica——比 Helvetica 早 60 年）
- 1950 年起 Tonhalle 音乐会海报系列 ("Musica Viva")：**"用视觉翻译音乐数学结构"**——形式追随内容
- 1958 起创办 New Graphic Design 杂志

**Vignelli (1931-2014)**
- Italian modernist, NYC 工作
- **American Airlines logo (1967, 用至 2013)** / NYC Subway signage (1972) / Washington Metro
- 核心哲学（原文）："semantically correct, syntactically consistent, pragmatically understandable" / "visually powerful, intellectually elegant, and above all **timeless**"
- **"Timeless" 是反 AI-feel 核心**：AI 生成 = contextual / Vignelli 风格 = timeless

**Why it matters** —— **Müller-Brockmann 的"用视觉翻译数学结构" + Vignelli 的"timeless" = 双重反 AI-feel 立场**。AI-feel 设计的"装饰性"vs 这两位的"结构性"是 21 世纪设计师应该学的对照。

## 2. 字体公司哲学三家对比

| 公司 | 哲学 | 字体命名风格 | 代表 |
|---|---|---|---|
| **Klim** (NZ, Kris Sowersby) | **字体是文化评论** | 学术化 | American Grotesk, Martina Plantijn, Die Grotesk |
| **Grilli Type** (Swiss) | **字体是瑞士传统延续** | 描述化 | GT Alpina, GT Standard |
| **Swiss Typefaces** (Swiss) | **字体是时代机器** | 性格化 | Black Mamba, BRRR, Vogy, Smog |

**原文摘录**：
- **Klim/American Grotesk**："in a time when truths, half-truths, lies, fiction, or entertainment seem **interchangeable**" = 字体命名是文化评论
- **Grilli Type**："the Swiss tradition"
- **Swiss Typefaces**："graphical machines, engineered to **capture the vibes of the now**" = 工业感、精确、可生成

**Why it matters** —— **Klim 把字体当文章写**（AI 写产品说明不写文化评论） / **Swiss Typefaces "vanguard visuals"** = **主动选边反 AI 中性化**

## 3. Pentagram IEEE 2026 Honors Ceremony

**项目**：IEEE 50 万+ 成员技术组织颁奖 / Pentagram 负责人 **Giorgia Lupi (data humanism 创始人)**
**2026 获奖者**：Jensen Huang (NVIDIA) / Luis von Ahn (Duolingo) / Marian Croak (Google)

**Pentagram 5 步方法**（原文）：
1. **数据深潜**："thousands of data points" → individual profiles
2. **分类**：award type / location / area of impact / institutional affiliation / conferences
3. **注入个人故事**：anecdotes / mentors / collaborators
4. **三个脚本**："three scripts and design directions"——**不是直接画，是先写 3 个版本脚本**
5. **选隐喻**：**"motherboard"** = 有意选的具体隐喻（不是泛泛"连接线"）

**关键哲学**（Lupi）："Data is the entry point to its design, but the outcome is a **hand-crafted visual story** that highlights the human connections essential to innovation."

**反 AI-feel 清单对照**：
- ❌ 不靠自动生成
- ❌ 不靠对称完美
- ❌ 不靠流行色
- ✅ 靠个人故事
- ✅ 靠手工艺感
- ✅ 靠具体隐喻

**Why it matters** —— **"Data is entry, hand-craft is outcome"** = Pentagram 2026 真实项目方法学。Corpus 的 Pentagram 综合主要讲历史结构，没这层 2026 实践细节。

## 5. shadcn Button 源码（apps/v4/registry/new-york-v4/ui/button.tsx）

**关键技术**：
- **CVA (class-variance-authority)**：变体管理 = design token 用 TypeScript 类型表达
- **Radix UI Slot**：组件组合 (asChild 模式)
- **data-slot attribute**：CSS + JS 精确选择

**6 variants** × **8 sizes** = **48 组合**（自动生成）：
- variants: default / destructive / outline / secondary / ghost / link
- sizes: default (h-9) / xs (h-6) / sm (h-8) / lg (h-10) + 4 icon-* (基于 4px 栅格)

**可访问性底线**（HIG 一致）：
- `aria-invalid:border-destructive` — 表单错误
- `focus-visible:ring-[3px]` — 键盘焦点（**Apple HIG 标准 3px**）
- `disabled:pointer-events-none disabled:opacity-50`

**暗色模式**：**写的时候就用 `dark:bg-input/30` 考虑，不是后处理**

**4 个核心设计模式**：
1. **Token 化的变体**——TypeScript 对象 = design token 类型化
2. **尺寸 + 变体 = 笛卡尔积**——6×8 = 48 组合自动生成
3. **data-attr 而非 class**——`data-variant="destructive"` 比 `class="destructive"` 更精确（CSS + JS 同选）
4. **暗色模式不是后处理**——`dark:` 前缀写时考虑

**Apple HIG 呼应**：
- `focus-visible:ring-[3px]` ← HIG 焦点环标准
- 暗色模式 + 高对比度变体 ← HIG "supply light and dark variants, and an increased contrast option"
- aria-invalid ← HIG Accessibility 章节

**Why it matters** —— **shadcn 不是"组件库"是"代码模板"**——它示范了 **design system 在 2026 怎么从 CSS class 升级到 TypeScript 类型 + data-attr**。这是 corpus 里 824 篇 v5+v6 没有的**当代 shadcn/MCP 时代**实操。

## How to apply
- 做**反 AI-feel 设计**时：学 Müller-Brockmann "翻译数学结构" + Vignelli "timeless"
- 做**字体选择**时：知道 Klim "文化评论" / Grilli "瑞士传统" / Swiss Typefaces "时代机器" 三种哲学
- 做**Pentagram-style 数据故事**时：用 Lupi 5 步法（数据深潜 → 分类 → 个人故事 → 3 脚本 → 选隐喻）
- 做**a11y 设计**时：学 A List Apart "design affects life and death" 立场
- 做**shadcn-style 组件**时：用 CVA + data-attr + dark: 前缀而非后处理
