# Design Anti-Portfolio — What the Learning Project Changed

Source: reflection after batches 001-025 and the critique of `APPLIED-DESIGN-JUDGMENT.md`.

This is not a success portfolio. It records wrong instincts that became less defensible after the corpus. The point is to keep the dents, because polished principles hide where learning actually happened.

## 1. I used to treat “AI design” as a visual defect. Now I think it is a memory defect.

Old instinct:

AI design looks bad because it overuses certain visible tropes: purple gradients, centered hero sections, glass cards, fake 3D blobs, generic SaaS dashboards, emoji icons, and soft shadows.

What changed:

Those tropes are symptoms. The deeper failure is that the design has no memory of why any form exists. It does not remember medium, production, institution, body, labor, public use, typography history, color physics, signage distance, licensing, print substrate, reading duration, or maintenance.

The bad gradient is not bad because purple is forbidden. It is bad because it has no contract with the product’s situation. It is a floating mood.

New rule:

When reviewing AI output, do not ask first “is this trope overused?” Ask “what human situation does this form remember?” If the answer is nothing, the design is amnesiac even if it avoids every known AI cliché.

## 2. I used to over-trust “clean minimalism”. Now I distrust clean screens that have no states.

Old instinct:

A clean screen with good spacing, one accent color, one primary action, and restrained type is probably better than a busy screen.

What changed:

Batch-017 and batch-024 make that too easy. Minimalism can be hostile when it erases affordance, error recovery, progress, context, or reading cues. Flat design inherits part of Swiss clarity, but it can also remove the very cues users need. Interaction design has temporal form; a static minimal screenshot often looks clean because it has hidden all the temporal obligations.

New rule:

Never praise a minimal UI until I have seen its loading, empty, error, partial success, disabled, undo, focus, and mobile states. A single clean frame is not a product; it is a costume test.

## 3. I used to think typography choice was mostly taste plus readability. Now I see it as infrastructure plus memory.

Old instinct:

Pick a good sans for UI, maybe a distinctive display face for headings, check contrast and hierarchy, avoid obvious defaults if brand matters.

What changed:

Typeface, font, glyph coverage, optical size, tracking, leading, alignment, OpenType features, licensing, foundry support, script direction, and print/digital conditions are not refinements after the style. They are the conditions under which the style can exist.

The corpus also changed the memory side: type can preserve a shipyard brush lettering, a public theater voice, a newspaper texture, a language’s reading norm, a screen technology’s compensation, or a civic signage tradition.

New rule:

A font choice must answer two questions: what work does it survive, and what memory does it carry? If it answers neither, it is just decoration.

## 4. I used to collapse “modernist” into one clean doctrine. Now I see several incompatible modernisms.

Old instinct:

Modernism means grid, sans-serif, clarity, restraint, less is more, form follows function.

What changed:

Pentagram governance, De Stijl grammar, Crouwel grid discipline, Vignelli systems, Rand business simplicity, Glaser civic warmth, Cassandre commercial compression, Bauhaus education, Swiss objectivity, and London Transport civic integration are not the same thing. They share some family resemblance, but they answer different pressures.

The danger is using “modernist” as a moodboard tag. That turns history into a costume rack.

New rule:

When invoking a tradition, name the mechanism: governance, grammar, signage, corporate memory, public clarity, production constraint, education method, commercial speed, or civic affection. Do not say “modernist” unless I can say which modernism and why.

## 5. I used to think “structure before surface” was almost universal. Now I think cultural voice can legitimately lead.

Old instinct:

Start with IA, taxonomy, hierarchy, task flow, wireframe. Then style.

What changed:

That is right for operational products, but insufficient for cultural institutions, editorial design, posters, campaigns, and some identities. Paula Scher’s Public Theater work is not loud after structure; loudness is part of the structure because the institution’s publicness is the message. Poetry Magazine’s covers preserve identity through repeated reinterpretation, not static IA.

New rule:

Ask whether the first user problem is orientation or recognition. If orientation, structure leads. If recognition, stance, or belonging, voice may lead — but it still has to become a rule system, not a mood.

## 6. I used to treat “honest UI writing” as a copywriting issue. Now I see it as a power issue.

Old instinct:

Good microcopy is clear, action-oriented, specific, and human. Avoid “Oops!” and internal jargon.

What changed:

The microcopy readings made the ethical layer harder to ignore. Cookie banners, unsubscribe flows, progress bars, password rules, AI output disclosures, and errors are all small negotiations of power. A cheerful tone can hide asymmetry more effectively than a cold tone.

New rule:

Before editing tone, identify what the user is being asked to give up: time, data, money, certainty, attention, control, privacy, or dignity. Then write the UI so that trade is visible.

## 7. I used to think “human-centered design” meant enough user research. Now I think it can still fail if the system boundary is wrong.

Old instinct:

Talk to users, test tasks, iterate, measure usability.

What changed:

The healthcare/eHealth sources are a useful correction. User feedback is not the whole system. Workflows, reimbursement, professional roles, literacy, access, regulation, racism, disability, organizational incentives, and support burden can defeat a screen that tested well in isolation.

New rule:

User research must include the system around the user. If a design makes an individual’s screen nicer while shifting burden to nurses, moderators, admins, translators, family members, or low-literacy users, it has not solved the design problem.

## 8. I used to think provenance was research hygiene. Now I think it is aesthetic hygiene.

Old instinct:

Track sources so claims are verifiable.

What changed:

The missing-source gap forced a sharper conclusion: hallucinated provenance produces hallucinated taste. If I pretend to have read a source, I do not just damage the bibliography; I contaminate the design judgment built on it. A moodboard with no lineage creates the same problem visually.

New rule:

A design claim should carry its evidence class: deep read, reference, duplicate, cross-language validation, missing, or inference. Taste without provenance becomes vibes with footnotes.

## 9. I used to think “beauty vs function” was a tired binary. Now I still think that — but with a caveat.

Old instinct:

Beauty can be functional. This is true and easy to say.

What changed:

Sagmeister’s work makes the positive case, but also exposes the danger: charisma can overwhelm evidence. Beauty can increase attention, care, and memory. It can also seduce the designer into thinking the argument is stronger than it is.

New rule:

Beauty is functional only if I can name what function it changes: comprehension, memory, willingness to engage, trust, pacing, perceived dignity, or emotional endurance. If I cannot name the function, I should stop using the phrase.

## 10. I used to think the output should be a handbook. Now I think the missing layer is adversarial self-review.

Old instinct:

Compress the corpus into principles, checklists, prompt templates, and modes so it becomes usable.

What changed:

The critique is right: a handbook is useful, but it can become sterile. Real taste needs conflict, regret, wrong turns, and cases where good principles lose. A perfect checklist may prevent obvious slop but cannot produce judgment under contradiction.

New rule:

For every new synthesis document, include at least one of these: a conflict, a changed belief, a failed application, or a case where a beloved principle should lose.

## Short version

The learning project did not teach “better visual style”. It taught distrust of style without memory.

The most important sentence remains:

> Bad AI design has no memory of medium, institution, production, reading, use, labor, or history.

The anti-portfolio version is harsher:

> My earlier mistake was trying to fix amnesia with prettier style rules.
