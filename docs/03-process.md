## 一、评审与批评

**批评四层**（评审前 30 秒在白板写"今天在哪一层"，砍掉 50% 无效会议）：个人偏好（"蓝色怪怪的"→追问数据）/ 工艺（"间距没对齐 8pt"→能否给 lint 规则）/ 历史（"iOS 早不用拟物了"→哪年哪本规范）/ 理论（"违反格式塔接近性"→原则优先级）。

**45 分钟会议结构**：对齐 5min（重述目标+persona）→ 作者讲"为什么这样选" 5min →（Round Robin 20min，每人"1 件有效的+1 个问题"——比自由发言快 3 倍且保护内向者）→ 分歧归层 10min → 行动闭环 5min。**会后 24h 发 recap：3 件改了、3 件不改及原因——没闭环的评审 = 没发生。48h action items 必关闭。**

**反馈三分类**：有规范/数据支撑→**Action**（WCAG/Material 就是答案，不需要理由）；品味/权衡→**Persuade**（必须给 1 句理由）；没看懂→**Clarify**（永远先于 Persuade）。

**三级评审严格分开**：Critique（设计师 3-6 人，owner=设计师）/ Stakeholder Review（≤10 人，owner=PM，对齐范围非抠图）/ Ship Review（≤5 人，Eng Lead 是闸门）。灾难场景：stakeholder 在 critique 会上大改交互。

**评审句式**（精选）："如果目标是转化率 3%→5%，这个 CTA 是帮助还是阻碍？"；"我注意到 X，可能导致 Y，因为 Z"；"这条是观察不是判断：CTA 对比度 3.2:1，AA 需要 4.5:1"；"内容 10 倍长这个布局还成立吗？"；"65 岁用户第一次用会发生什么？"；"我猜这个选择背后的考虑是……对吗？"

**公开 vs 私下**：工艺/规范/数据问题→公开；能力/努力问题→私下；纯品味且不确定→闭嘴。公开批评要署名、可验证、给改进路径。

**制度化样本**：Linear "Quality Wednesdays"——每周三每人必须 find and fix 一个质量 defect，两年 1000+ 小修复。Linear/Stripe 共同流程：先写"为什么值得做"的文章再画界面（"Output isn't design"）；新功能先问"现有 token+组件能否组合出来"——这一步把 brief 推到 60% 完成度。

**AI 套路审稿补充**：通篇 Inter→加衬线对比；文案"赋能/打造/闭环/生态"→产品名+动词；每功能配图标→只给核心 3 个；4 列 feature grid→1 列深挖。原则：AI 给 80 分平均值，找出"哪个 100 分产品和你一样"朝它改。

## 二、用户研究

- **5 用户原则**：N×(1-(1-0.31)^n)——5 人≈85% 问题；**5 人 × 3 轮 > 15 人 × 1 轮**（每轮之间重新设计）。
- 三种可用性测试：Moderated（5-8 人，可追问）/ Unmoderated（20-100+，任务须极清晰）/ Guerrilla（1 天 30 个数据点）。发版前组合：Moderated 5 + Unmoderated 30+。任务写成故事不写步骤（"打印机显示 5200 错误，怎么解决？"）。
- **访谈 5 禁问**："你会用吗"（99% 答会）/"好看吗"/"如果有 X 你会不会用"/"为什么买"（事后合理化）/"愿付多少钱"。正确问法全部指向**过去行为**："上次遇到是什么时候？做了什么？哪一步最麻烦？"5-8 人到饱和。
- **Card Sort**：30-60 张卡；开放式 ≥15 人；70%+ 用户分到一起的卡形成新 IA。陷阱：卡片用产品术语而非用户语言。
- **Tree Test**：30+ 用户；成功率 <60% 严重 / 60-80% 优化标签 / >80% 达标；**看错点位置比成功率更有价值**；NN/g：首击正确 → >87% 任务成功。
- Diary Study：高频 1-2 周低频 3-4 周；**多招 20-30%（流失 30%）**。问卷：95% 置信 ±5% 至少 400 份；反向题抓"全选 5"。
- Affinity：每人 30-50 张便利贴 → **沉默聚类** → 命名（动词+名词）→ 红点投票。
- 研究简报结构：发现 → 证据（"5/8 用户在第 3 步放弃；原话'我看不到总金额'"）→ 建议 → 影响 → 优先级 → 负责人。
- AI 边界：Synthetic Users 只用于早期发散；自动聚类省 80% 整理时间；不能替代真实环境观察。

## 三、A/B 测试

- 样本量速算：`n = 16·p·(1-p)/MDE²`；基线 5% + 相对 MDE 10% → ~75k/变体。最低门槛：每变体 ≥50 曝光 + ≥5 转化。
- **可测/不可测黄金法则**：能一句话描述的 UI 差异→可测；需一整段解释→不可测（IA 重组、核心交互范式、0→1 新功能——用原型+访谈）。
- 流量决策树：<10k/天只测 1 变量；10k-100k 可 A/B/n（**显著性阈值除以 n**，Bonferroni）；>100k 才 MVT。
- **5 反模式**：偷看（偷看 100 次假阳性 5%→40%+）；p-hacking（预先钉死 1 主指标+护栏指标）；多重比较；不跑 A/A 验证工具；一锤定音。
- **指标金字塔**：设计指标→行为指标→业务指标，传导递减；**永远不要直接看北极星**（噪声大到跑一年不显著）；必配护栏指标（退款率/客服工单/加载时长）。
- 判定脚本：达样本量 + 置信区间不跨 0 + 护栏未恶化 + 跑完 1 个完整业务周期（7-14 天）→ 胜出；区间含 0 → 归档，不追加跑不换指标。
- 6/7 实验会失败，失败文档比胜出方案值钱；**A/B 只告诉"什么有效"不告诉"为什么"——永远配 5 个用户访谈**。

## 四、信息架构

- 四大系统（Rosenfeld & Morville）：组织/标签/导航/搜索。5 种组织模型选型：B2C 内容站=层级+分面；SaaS=轮辐+数据库（**Linear/Notion/Slack 几乎全用 Hub-and-Spoke**）；知识库=超文本+搜索。
- 标签策略：高频入口用受控词表；长尾 UGC 用 folksonomy+审核；搜索层用**同义词环**把"账号/账户/Account"归一。
- **80% 的"搜索不准"源自标签错乱/类目错位**——修搜索是给破屋子装报警器。顶级产品都提供"层级浏览+搜索跳转"双通道，剥夺任一条丢 30%+ 用户。
- 案例锚点：Stripe Docs **任务优先于产品**（"Accept payments" 作顶层）；GOV.UK 按**生活事件**（Having a baby）而非政府部门组织；SharePoint 旧版=IT 拍板类目与业务心智脱节，全员用全局搜索绕过 IA——IA 失败的典型结局。
- Rosenfeld："Stop Redesigning, Start Tuning"——重设计前先做 Tree Test 基线；上线后用搜索日志持续调优。

## 五、服务设计

- Stickdorn 5 原则中最常被忘的是 **Evidencing（物证化）**：订单确认页、进度条让无形服务被看见。
- **Journey Map 管"哪一段疼"，Blueprint 管"哪一层哪个角色失职"**。Journey：一张图只服务一个 persona；情绪曲线找低谷；痛点=预期与实际的差距，不是 bug。Blueprint：4 层（Customer/Front-stage/Back-stage/Support）+ 3 线（尤其 Line of Visibility）；每层之间标 fail point；**每个动作必须有 owner 否则是装饰品**。
- 案例锚点：Mayo Clinic 用协调员把 3 周就诊压缩到 3 天；GOV.UK 800+ 网站整合为 1 入口 + 14 条 Service Standard 违者下架；反例=手机银行消灭排队但放款仍 7 天——**前台优化+后台失配=体验更差**。
- 触点是时间链非孤立点：就诊触点链至少 12 个，任一断点毁全链。

## 六、商业、定价与合同

- **三种咨询活法**：Pentagram（合伙人即股东即主创，无 CEO/销售，收入平分）；Lippincott（卖 "measurable growth" 不卖 logo，**向 CEO 汇报不向 CMO 汇报**）；MetaDesign（教训：人力资本不锁住会被并购方抽走）。
- **定价**：时薪 $75-300 是天花板最低的模式（与客户利益对立）；价值定价可达 5-20 倍。组合：Discovery 固定价 + 执行月费 + Retainer。**3-3-2 报价法：3 万 / 3 周 / 2 轮修改**；永远 30-50% 首付。
- **合同 6 必写**：IP 归属 + **设计师保留作品集展示权**（漏写可被禁止展示）；修改轮数 2-3 轮；kill fee 按阶段 30/50/75%；付款时间表；交付物清单（明确不含第三方授权）；NDA 违约金。8 条定价反模式：时薪当默认 / 不写轮数 / 无 kill fee / discovery 与执行不分 / 朋友价混市场价……
- **创业谱系**：Freelance（99% 起步首选）→ Micro Studio（2-5 人）→ Partnership → Design-led SaaS（极高风险）→ 课程/内容。规律：资本越少自由度越高、收入天花板越低。Design-led SaaS 原则：**opinionated > configurable**（Linear $1.8B 的核心）、速度 <100ms、键盘 90%。
- 个人品牌金字塔：长文 → Newsletter → 书 → 课程 → 高客单咨询；保留 20% 精力做声誉资产。

## 七、字体授权与法律

- **4 大授权类型**：SIL OFL（免费商用可嵌入可改，唯一禁令是单独卖字体本身；Inter/IBM Plex/Noto/Fraunces）；Adobe Fonts（**仅订阅期间有效**，取消须下架字体）；Monotype 系（Desktop/Web/App/Broadcast 分开卖；**系统预装 ≠ web 商用免费**——Arial 嵌入 @font-face 仍要授权）；独立 foundry（Klim/Commercial Type/Grilli，一次买断，Söhne 全家 $2000-5000）。
- **7 个授权陷阱**：桌面与 web 分开买；按域名计；按月活分级（100 万 UV 要 enterprise $5000+）；app 按下载量（千万下载可能 $50,000+）；多数商业授权禁修改/subset；**很多字体禁用于 logo**（Söhne logo license $5000-10000）。真实案例：初创未授权用 Helvetica Neue 被索赔 $50,000。
- 版权：美国个人死后 70 年/雇佣作品 95-120 年；中国死后 50 年。logo 必注册商标（10 年可续，3 年不用可撤销）。Google Fonts CDN 有 GDPR IP 风险——self-host 或 Bunny Fonts。

## 八、职业与教学

- **招聘者 5 分钟规则**：作品集 = 钩子 + 2-3 个深度 case（不是 8 个浅的）+ 真实影响数据。新人作品三选一：标注 unsolicited 的重设计 / **非营利志愿项目（最推荐，有真实用户）** / 设计挑战赛。Take-home 潜规则：写明"没做什么和为什么"+ 反思段落得分高 3 倍。56% 成功 UX 转行者来自无关行业；顶院真正价值=200 人高密度反馈圈。
- AI 时代 craft 价值排序（Figma 调研）：视觉打磨 58% > 问题拆解 47% > 直观 UX 36% > 跨产品一致性 15%。
- **教学**：黄金比例 30% 讲 + 70% 做；永不连续讲 >20 分钟；反馈 <24h、格式"3 好+3 改进+1 练习"；deliberate practice=难度高于能力 10-20%。**价格 = 完成率**：<$100 → <10% 完成；$2000+ → 70-90%。课程承诺要具体（"学完能做 SaaS 落地页"），过度承诺就业率有法律风险。
