# 设计师创业 / 微型工作室：执行手册

## 一句话总结
**设计师"以后要做设计"有 6 条路——不是只有"开一家 Pentagram"**。Freelance → 微型工作室 → 合伙制 → design-led SaaS → 课程 / 内容 → 设计投资，每条路资本、风险、回报、节奏完全不同。**选错路比努力重要 10 倍**。

## 1. 6 种起步模式对比

| 模式 | 启动成本 | 月收入潜力 | 自由度 | 失败风险 | 适合谁 |
|---|---|---|---|---|---|
| **Freelance（自由职业）** | $0 | $3k-$30k | 高 | 低 | 想接项目不想管团队 |
| **Micro Studio（2-5 人）** | $5k-$20k | $20k-$200k | 中 | 中 | 想做更大的项目 |
| **Partnership（合伙制）** | $20k-$100k | $100k-$1M+ | 低 | 高 | 像 Pentagram / Manual 一样做品牌 |
| **Design-led SaaS** | $50k-$500k | $100k-$10M+ | 中 | 极高 | 想 scale 想做被动收入 |
| **Courses / Content（课程 / 内容）** | $5k-$50k | $10k-$500k+ | 中 | 中 | 想输出 + 教育 |
| **Design Investment（设计投资）** | $100k+ | 不可预期 | 极低 | 高 | 已有资金 / 关系网 |

**学到的**：
- **资本越少 → 自由度越高 → 收入天花板越低**
- **资本越多 → 自由度越低 → 收入天花板越高**
- **99% 设计师起步选 freelance**——> 0 风险试错

## 2. 模式 1: Freelance（自由职业）

### 启动清单
- ✓ 作品集网站（个人域名 + 自建站）
- ✓ LinkedIn / Twitter / Instagram 公开身份
- ✓ 1-2 个免费试做（降低客户首次合作风险）
- ✓ Stripe 账号 + 银行账户
- ✓ 简单合同模板（Work for Hire + 6 必写条款）

### 5 步接单流
1. **冷邮件 / 介绍**——> 第一批客户来自朋友 / 朋友的朋友
2. **Discovery call**——> 30 分钟了解客户需求 + 报价
3. **签合同 + 首付 50%**——> 永远不免费开始
4. **项目执行**——> 视频/异步沟通，避免 scope creep
5. **尾款 + 收尾**——> 提供 2 周免费修改，之后按小时

### 报价公式
- **新手**：$50-$100 / 小时 或 $3k-$10k / 项目
- **中级**：$100-$200 / 小时 或 $10k-$30k / 项目
- **高级**：$200-$500 / 小时 或 $30k-$100k / 项目
- **专家**：$500+ / 小时 或 $100k+ / 项目

### 4 个反模式
1. **时薪报价当默认**——> 应价值定价
2. **不写修改轮数**——> 客户无限改稿
3. **不收首付**——> 客户拿了稿跑
4. **不挑客户**——> 烂客户消耗你，坏口碑传染

## 3. 模式 2: Micro Studio（2-5 人工作室）

### 启动清单
- ✓ 2-3 个互补搭档（设计 + 工程 + 战略）
- ✓ 合伙协议（股权 / 决策 / 退出）
- ✓ 共享工作空间或全远程
- ✓ 共享客户池（不抢单）
- ✓ 工作室品牌（命名 + 视觉）

### 5 大挑战
1. **客户分配**——> 谁接什么客户？
2. **利润分配**——> 贡献 vs 股权？
3. **决策机制**——> 一票否决？多数？
4. **招聘**——> 第 1 个雇员最难
5. **现金流**——> 大项目 6 个月不开张

### 5 个成功案例
- **Manual**（旧金山）——> 5 人设计工作室，年收 $1M+
- **Locomotive**（蒙特利尔）——> 8 人 web 工作室
- **Mule Design**（波特兰）——> 5 人品牌工作室
- **Frank & Hank**（远程）——> 2 人品牌工作室
- **Resn**（新西兰）——> 10+ 人创意工作室（实验性）

## 4. 模式 3: Partnership（合伙制 / 像 Pentagram）

### Pentagram 模式
- **5-25 合伙人平等**（no CEO, no boss）
- **每个合伙人独立带客户**——> 共享办公室 + 后台
- **利润按合伙人分成**（非按股权）
- **新合伙人需现有全票通过**——> 严格
- **退出合伙人 18 个月内不能开类似公司**

### 学到的 5 个核心
1. **没有传统层级**——> 合伙人是平等的伙伴
2. **共享品牌**——> 客户认可的是 "Pentagram" 不是个人
3. **共享后台**——> 行政、财务、PR 共担
4. **独立业务 + 共享成本**——> 自由 vs 杠杆的平衡
5. **退出机制严格**——> 18 个月不能挖墙脚

### 适配性
- **适合**：资深设计师，已有客户，资本 $100k+
- **不适合**：新手设计师（资历不够带客户）
- **真实案例**：Pentagram / Lippincott（5 人变 25 人）/ MetaDesign

## 5. 模式 4: Design-led SaaS

### 5 个最成功的 design-led SaaS

#### (1) **Figma**（2012 成立，2022 上市）
- **起源**：设计协作工具
- **关键设计决策**：design + engineering 共享
- **资本**：Dylan Field 个人 + YC + Greylock + Sequoia
- **退出**：Adobe 200 亿美元收购（2023 取消）
- **学到的**：**从"设计师做给设计师"到"所有设计相关人都能用"**——> scale 路径

#### (2) **Linear**（2019 成立，2024 估值 18 亿）
- **起源**：issue tracking 工具
- **关键设计决策**：opinionated + 极简 + 速度
- **资本**：创始人自筹 + a16z + Sequoia
- **学到的**：**"opinionated > configurable" 是设计产品能赢的核心**——> 不学 Jira 100 配置

#### (3) **Vercel**（2015 成立，2024 估值 30 亿+）
- **起源**：Next.js 团队的部署平台
- **关键设计决策**：开发者体验优先 + 极简
- **学到的**：**"做给开发者也用设计语言"是 SaaS 新模式**

#### (4) **Mailbrew**（已关闭）
- **起源**：邮件摘要工具
- **关键失败**：Niche 太窄
- **学到的**：**Niche 产品难 scale**——> 太专注 = 用户少

#### (5) **Webflow**（2013 成立，2022 估值 40 亿）
- **起源**：可视化建站
- **学到的**：**"给非工程师做" 是巨大市场**——> 1% 程序员 + 99% 普通人

### 5 大 design-led SaaS 原则
1. **Opinionated > Configurable**——> 默认值就是最佳实践
2. **极简 ≠ 简单**——> 极简是把复杂度藏在背后
3. **速度是核心**——> 每次操作 < 100ms
4. **键盘党友好**——> 90% 操作用键盘
5. **设计语言一致**——> 整个产品一个声音

## 6. 模式 5: Courses / Content（课程 / 内容）

### 5 种变现方式
1. **付费课程**（Domestika / Skillshare / 自建）
2. **Newsletter**（Substack / Beehiiv）
3. **YouTube**（视频 + 广告 + 会员）
4. **播客**（赞助 + 会员）
5. **模板 / 资源**（Figma 模板 / 字体 / 资产）

### 真实案例
- **Refactoring UI**（$249 课程）——> $1M+ ARR
- **Design + Code**（Meng To）——> $50k+/月
- **Sketch Together**（$200 课程）——> 长尾收入
- **Maggie Appleton**（独立研究者）——> 影响力 + 演讲邀请
- **Josh Comeau**（CSS 课程）——> $200k+/年
- **Wes Bos**（JavaScript 30）——> $500k+/年

### 4 个关键决策
1. **选主题**——> 你比谁强？
2. **选平台**——> 自建 vs 第三方
3. **选定价**——> 免费（影响力）vs $50-$500（收入）vs $1000+（高端）
4. **持续输出**——> 季度更新 vs 月度 vs 每周

### 5 个反模式
1. **录课不更新**——> 课程 6 个月就过时
2. **定价太低**——> $20 课程被认为不专业
3. **靠课程单一收入**——> 续费率 < 10%，需不断获客
4. **不建社区**——> 学生学完就走，没有口碑
5. **盗版忽视**——> 课程内容必加密 / 直播

## 7. 模式 6: Design Investment（设计投资）

### 5 种投资方式
1. **设计咨询 + 股权**——> 给初创公司设计换股权
2. **工作室 + 加速器**——> 像 Designer Fund 投资 + 辅导
3. **设计收购**——> 买有潜力的小工作室
4. **天使投资**——> 设计师自己投设计师
5. **运营设计社区**——> 影响力 + 关系网

### 真实案例
- **Designer Fund** ——> 投资设计师 + 设计师办的初创
- **Notion Fund** ——> 给设计师 + 写作者资助
- **Figma Config 投资**——> Figma 投给设计生态
- **Bryan Lovin (Designer, ex-Twitter)**——> 投资 + 咨询

### 适配性
- **适合**：有资金 / 关系网 / 经验的设计师
- **不适合**：新手（缺钱 / 缺关系）

## 8. 5 大失败模式

### 失败 1: 低价陷阱
- **症状**：接 $500 1 页设计，单价太低养活不了自己
- **修正**：永远报"3-3-2"——> 3 万起 3 周交付 2 轮修改
- **学到的**：**单价不涨 = 永远在做烂项目**

### 失败 2: Scope Creep
- **症状**：客户说"再改一下"，永无止境
- **修正**：写明修改轮数 + 超额按时收费
- **学到的**：**不设边界 = 客户把你当下属**

### 失败 3: 现金断流
- **症状**：大项目 6 个月不开张，付不起房租
- **修正**：永远 30% 首付 + 维护月费
- **学到的**：**单次大项目 = 现金流风险**

### 失败 4: 招错人
- **症状**：第 1 个雇员搞砸，团队崩
- **修正**：先用 freelance 3-6 个月测试，再 offer 长期
- **学到的**：**招人比解雇容易 10 倍**

### 失败 5: 品牌混乱
- **症状**：今天做品牌，明天做电商，后天做 SaaS
- **修正**：聚焦 1-2 个垂直
- **学到的**：**专业 = 溢价**

## 9. 5 步从 Freelance → Micro Studio

### Step 1: 独立接单 12-18 个月
- 学会报价 / 合同 / 客户管理
- 攒 10-20 个满意客户
- 存款 6 个月生活费

### Step 2: 找 1 个搭档（互补）
- 你是设计 → 找工程
- 你是品牌 → 找 UI
- 你是策略 → 找执行

### Step 3: 签合伙协议
- 股权：50/50 或 60/40
- 决策：一票否决 or 多数
- 利润：按月分配
- 退出：3-6 个月通知期

### Step 4: 共用品牌
- 命名：你的名字 / 共同名字
- 域名 + 邮箱
- 共享 Slack / Notion / Figma

### Step 5: 持续 3-5 年
- 不断招聘（每年 1-2 个）
- 品牌建设（作品集 / 演讲 / 出版）
- 产品化（从项目 → 模板 → 课程 → SaaS）

## 10. 个人品牌建设 8 步

1. **个人网站**——> 自建（next.js / Astro / 11ty），不用 Squarespace
2. **公开 Twitter / LinkedIn**——> 每周 3-5 条关于设计思考
3. **公开案例**——> 长文 + 截图 + 视频
4. **公开演讲**——> 申请 Config / Layers / Awwwards
5. **出版**——> 写书 / 杂志专栏 / Newsletter
6. **社区贡献**——> 维护开源 / 写工具
7. **导师**——> 教 1-2 个初级设计师
8. **持续 3-5 年**——> 个人品牌 = 时间复利

**真实案例**：
- **Linear 创始团队**——> 持续 Twitter 输出 → 估值 $1.8B
- **Jason Yuan (Figma)**——> 个人项目 → 大厂 offer
- **Rauno Freiberg (Vercel)**——> 开源 UI 库 → 影响力

## 11. 7 个财务模型

### 模型 1: 项目制（Freelance 标配）
- 单项目 $5k-$100k
- 月接 1-3 个项目
- 月收入 $5k-$30k

### 模型 2: 月费制（设计顾问）
- 月费 $5k-$20k / 客户
- 同时服务 2-5 个客户
- 月收入 $10k-$100k

### 模型 3: 产品化（模板 / 课程 / SaaS）
- 一次开发，多次销售
- 月收入 $5k-$500k
- 后期被动收入为主

### 模型 4: 股权制（设计 + 股权）
- 1-2 个项目换 0.5-5% 股权
- 长期持有
- 退出收益（$100k-$10M+）

### 模型 5: 合伙制（Pentagram 模式）
- 利润分给合伙人
- 月收入 $50k-$500k（视合伙人数量）

### 模型 6: 雇佣（4 大 + 创业公司）
- 年薪 $80k-$300k + 期权
- 稳定但天花板有限

### 模型 7: 投资 + 咨询混合
- 投资初创 + 兼任设计顾问
- 月收入不可预期
- 长期价值高

## 12. 7 个常见反模式（必避）

### 反模式 1: 时薪报价
**修正**：价值定价（按项目 / 按月 / 按股权）

### 反模式 2: 接烂客户
**修正**：3 个问题筛客户（预算 / 决策权 / 时间表）

### 反模式 3: 不写合同
**修正**：永远 Work for Hire + 6 必写条款

### 反模式 4: 单一定价
**修正**：3-3-2 报价法（3 万 / 3 周 / 2 轮修改）

### 反模式 5: 客户免费修改
**修正**：2 轮免费，之后按 $200/小时

### 反模式 6: 招人太快
**修正**：freelance 3-6 月测试，再 offer

### 反模式 7: 不建品牌
**修正**：每周 Twitter / 季度写作 / 年度演讲
