ဆောင်ရွက်စေသည်။ ယင်းမှာ အသုံးပြုသူများအား စီစဉ်ဆောင်ရွက်ပေးထားပြီး ဖြစ်သော ပါဝင်ဝန်ဆောင်မှု များကို ကြည့်ရှုရုံ သက်သက် ကန့် သတ်ထားသော websiter များ နှင့် နှိုင်းယှဉ်မှုပြုခြင်း ဖြစ်သည်။ ဥပမာအနေနှင့် ပါဝင်ဖော်ပြနိုင်မှု နှင့် ထည့်သွင်းသုံးစွဲနိုင်သော လုပ်ငန်းဆောင်တာများကို စံပြုထိန်းချုပ်နိုင်ခဲ့လျှင် Microsoft မှ PC ဈေးကွက်ကို လွှမ်းမိုးထားသကဲ့သို့သော ဈေးကွက်အင်အားကို Netscape မှ ရရှိပါလိမ့် မည်။ ကားများ၏ ပုံစံမူရင်းမှာ "မြင်းမဲ့ ရထားလုံး" ဖြစ်သကဲ့သို့ ရှိရင်းစွဲကို ထပ်ပေါင်း၍ Netscape မှ desktop နေရာတွင် အစားထိုးရန် " webtop " သို့ အဆင့်တိုးလာခဲ့သည်။ ထို့နောက် ၎င်း webtop ကို သတင်း အချက်အလက်များ၊ အထူးပြု ရေးဆွဲထားသော လုပ်ငန်းဆောင်တာငယ်များ အသစ်ဖြည့်စွမ်း၍ Netscape ၏ server များကို ဝယ်ယူမည့်သူများ ပိုမို များပြားလာအောင်စီမံသည်။ အတိုချုပ်ဆိုရလျှင် Netscape မှ software များကို အသစ်တီထွင်၊ မကြာခဏ တိုးမြှင့် ပြောင်းလဲ၍ သုံးစွဲသူများထံသို့ ဖြန့်ဖြူးရောင်းချရန်ကို အဓိကထားခဲ့သည်။ O’Reilly က ၎င်းကို google ကဲ့သို့သော software ထုတ်လုပ်မှုများပေါ်တွင် အဓိကမထားပဲ browser သက်သက်အဖြစ် web ရေးသားသူများ ဆိုဒ်တစ်ခုနှင့်တစ်ခုအကြား ပြုလုပ် ထားသော ချိတ်ဆက်မှုများ အဆင်ပြေစေရန် ဝန်ဆောင်မှုပေးခြင်းကိုသာ အဓိက အာရုံပြုနေ သော ကုမ္ပဏီမျိုးနှင့် နှိုင်းယှဉ်ပြခဲ့သည်။ google မှ သူ၏ စီစဉ်ရေးဆွဲထားသည့် ထုတ်လုပ်မှုများကိုသာ ဆောင်ရွက်နေသော software များနှင့် မတူသည်မှာ ၎င်းဝန်ဆောင်မှုများသည် အစဉ်တစိုက် အသစ်အသစ် ပြုပြင်နေလေ့ရှိ ပြီး၊ "မရပ်မနားသောbeta " လုပ်ငန်းစဉ် ဟုလည်းခေါ်သည်။ Britannica အွန်လိုင်း စွယ်စုံ ကျမ်းနှင့် wikipedia တို့၏ တူညီသော ကွဲပြားခြားနားချက်မှာ Britannica အနေ နှင့် ဆောင်းပါးများဖန်တီးနိုင်ရန်နှင့် ပုံမှန်ရိုက်နှိပ်ထုတ်ဝေနိုင်ရန်အတွက် ကျွမ်းကျင်သူ ပညာရှင် များကို အားထားမှုပြုနေရချိန်တွင် Wikipedia မှာ အမည်မဖော်လိုသော အသုံးပြုသူတို့မှ ပုံမှန်နှင့် လျင်မြန်စွာ ထည့်ဝင်ရေးသားလာမှုတို့ကို အားထားပြုပါသည်။ Wikipedia မှ ကျွမ်း ကျင်သူများအပေါ်တွင် အခြေခံခြင်းမဟုတ်သော်လည်း ဖွင့်လှစ်ရေးသားခွင့်ပြုသော software ဆိုရိုးစကား ဖြစ်သည့် " မျက်လုံးများ အလုံအလောက်ရှိထား၍ အမှောင့်အဖျက်များ ကို တိမ်မြုပ်စေခြင်း" ဟူသည့်စကားနှင့်
