# 人类设计美学百科全书（最终综合版）

> 9 轮研究 · 60+ 来源 · 横跨东西方 5 个世纪的设计智慧
> 适用：网页 / PPT / 海报 / App / 程序 / 品牌 / 数据可视化

## 总纲：好设计的五个共同真理

跨所有大师、流派、时代，5 个反复出现的主题：

### 1. **Less is more — 删除是核心**
- Mies van der Rohe "Less is more"
- Rams "Weniger, aber besser"（少而更好）
- Muji 手绘原型（避免不必要细节）
- Tufte "Data-ink ratio"（删除非数据墨水）
- Vignelli "Simplicity, clarity, timelessness"
- 安藤忠雄"俳句效应"（强调 nothingness）
- **AI 缺的是"删掉的勇气"** — 这是 AI 设计最深的病灶

### 2. **Form follows function — 功能诚实**
- Bauhaus "form follows function"
- Rams 第 5 诫"Unobtrusive"（产品是工具）
- 路易斯·康"what does the building want to be?"
- 诚实材料：木头不假装大理石
- **AI 错把"装饰=功能"，加渐变/光晕/阴影让功能被掩盖**

### 3. **Restraint with intention — 克制但有主见**
- 日本 wabi-sabi "nothing is perfect"
- Shibui 七元素：克制 / 含蓄 / 谦逊 / 自然 / 日常 / 不完美 / 静默
- 瑞士风格"objective presentation"
- 加州 Mission 家具派的"民主设计"
- **AI 输出"安全"= 没有主见 = 立刻被识别**

### 4. **Asymmetric balance — 不对称的活力**
- De Stijl "aesthetic balance by use of opposition"
- 瑞士风格核心原则
- 摄影 rule of thirds（不是居中）
- 构图的 movement（眼睛要移动）
- **AI 喜欢对称 = 模板感 = 廉价**

### 5. **Truth to materials — 诚实表达**
- Kahn "I like an Arch"（对砖说话）
- 阿尔托"humanize architecture through unorthodox handling of form and materials"
- Rams 第 6 诫"Honest"（不夸大）
- Eames 湿胶合板"做中学"
- **AI 输出的"完美"= 不诚实 = 反而显得假**

## 第二篇：色彩（综合所有大师）

### 色环模型
- **RYB（传统）**：红黄蓝，Itten / De Stijl 用
- **RGB（加色）**：屏幕
- **CMYK（减色）**：印刷
- Itten 12 色立体（Hölzel 色环基础上）

### Itten 七对比（重要！）
一页最多用 1-2 种：
1. **色相 Hue**
2. **明度 Value**
3. **冷暖 Temperature**
4. **互补 Complement**
5. **同时 Simultaneous**
6. **饱和度 Saturation**
7. **面积 Extension**

### Albers 警示
- "color deceives continually"
- "we do not see colors as they really are"
- **永远要看实际应用效果，不要相信色卡**

### 心理与文化
- 红：紧迫 / 冲动 / 跨文化愤怒
- 蓝：信任 / 放松 / 理性
- 黄：快乐 / 温暖
- 绿：环保 / 可靠
- 紫：复杂 / 创意
- 暖色 → 冲动购买
- 冷色 → 理性决策
- 中英审美差异大
- 90 秒 62-90% 判断来自色彩

### 系统化配色
- 8-10 阶中性灰
- 1 个主色（≤5% 用量）
- Radix 12 阶：1-2 背景 / 3-5 组件底 / 6-8 边框 / 9-10 实色 / 11-12 文字
- 选带温度的灰：mauve / slate / sage / sand / olive
- 永不纯黑 #000，用 #0F0F0F
- WCAG：4.5:1 正文 / 3:1 大字

### 反 AI 套路
- 紫粉渐变 (#8B5CF6→#EC4899)
- 荧光色
- 饱和 blue-500 满铺
- AI 默认 palette
- 紫色被 AI 错当成"科技感"

## 第三篇：字体（综合所有大师）

### 字体史时间线
- 15c Garamond 老式
- 18c Caslon / Baskerville 过渡
- Late 18c Bodoni / Didot 现代（Didone）
- 19c Slab Serif
- 1890s Akzidenz-Grotesk
- 1920s Futura 几何
- 1928 Gill Sans 人文
- 1957 Helvetica / Univers 新怪诞
- 1958 Optima 人文 sans
- 2017 Inter 屏显优化

### 大师选择
- **正文 UI**：Inter / Geist / Söhne / SF Pro
- **正文长阅读**：Tiempos / Charter / Source Serif
- **标题 Display**：GT Sectra / GT Super / Migra
- **数据/代码**：JetBrains Mono / Berkeley Mono
- **避免**：Times New Roman / Calibri / 紫粉渐变字体 / Comi c Sans

### 关键度量
- Baseline / Ascender / Descender / X-height / Cap-height / Em-square
- **X-height 决定屏显可读性**
- 调高 x-height → 屏显友好
- 小尺寸用光学小号字（text face），display 用 display face

### 排版原则
- 字号比例：1.125 / 1.2 / 1.333
- 推荐尺度 12/14/16/20/24/32/48/64
- 正文 14-16px，标题与正文 ≥1.5× 差
- 行宽 45-75 字符（理想 66ch）
- 行高 正文 1.5-1.7，标题 1.05-1.2
- Display 大字负字距 -0.02em
- 大写字距 +0.05em
- 数据用 tabular figures
- Sentence case，不用 Title Case
- ALL CAPS 仅限 label/nav/CTA
- Bayer 1925 取消大写（理由：更人性）
- 2-3 个 family 上限

## 第四篇：布局与网格

### Swiss Style 原则
- 数学网格
- 左对齐 ragged right（永远）
- 不对称布局
- 客观表达
- "form follows function"
- 字体：Akzidenz-Grotesk / Univers / Helvetica

### 12 列响应式栅格
- 间距 4/8 倍数：4/8/12/16/24/32/48/64/96/128
- section 间距 ≥ 内部 2×
- 内容与边缘 ≥ 32px
- 重要元素偏左/偏上（F-pattern）
- 减少 1px border，用间距+色差+阴影代替
- 圆角统一：8 卡片 / 4 按钮 / 999 头像

### 圆角哲学
- 8px 卡片
- 4px 按钮
- 999px 头像
- 不要混用

## 第五篇：视觉层级（5 工具优先级）

1. **Size 大小**（最大差异）
2. **Weight 字重**
3. **Color 颜色**（gray-900 而非纯黑）
4. **Position 位置**（左上看优先）
5. **Whitespace 留白**（不是空，是关系）

### 原则
- De-emphasize to emphasize
- 眯眼测试
- 一页最多 3 个 size
- 一页最多 1-2 种 Itten 对比

### Gestalt 完形
1. Proximity 接近
2. Similarity 相似
3. Closure 闭合
4. Symmetry 对称
5. Continuity 连续
6. Common fate 共同命运
7. Figure-Ground 图形-背景

### Von Restorff 隔离效应
- 唯一不同 = 最重要
- 一页只 1 个显著元素

## 第六篇：UX 定律速查

| 定律 | 公式 | 应用 |
|------|------|------|
| **Hick** | T = b·log₂(n+1) | 选项越少越好；二分搜索式分步 |
| **Fitts** | MT = a + b·log₂(2D/W) | 按钮大、关键按钮在边缘 |
| **Miller** | 7±2 块 | 导航≤7、菜单≤7、流程≤7 |
| **Jakob** | 用户喜欢熟悉的模式 | 跟随约定 |
| **Von Restorff** | 唯一不同 = 重要 | 一页 1 个 CTA |
| **序列位置** | 头尾记得牢 | 重要项首末 |
| **Fitts edge** | 边缘"无限大" | 关键按钮放屏幕边角 |
| **Tesler** | 复杂度守恒 | 暴露而非隐藏 |
| **Parkinson** | 任务=占满时间 | 设限制 |

### 关键 UX 原则
- Affordances（感知到的属性）
- Signifiers（动作指示）
- Mapping（控制-效果对应）
- Feedback（立即响应）
- Constraints（约束）

## 第七篇：动画与动效

### Disney 12 原则（用于 UI）
1. Squash and stretch（点击反馈）
2. **Anticipation 预备**（hover 提示）
3. **Staging 舞台化**（动效引注意）
4. Pose to pose（关键帧）
5. **Follow through**（多元素错位结束）
6. **Slow in and slow out**（ease 缓动）
7. **Arc**（自然弧线）
8. Secondary action（次要动作补充）
9. **Timing**（节奏）
10. **Exaggeration**（反馈足够大）
11. Solid drawing（3D 感）
12. **Appeal**（动效有魅力）

### 缓动
- linear = 机械
- ease-out = 启动快，结尾慢（推荐用于进入）
- ease-in = 启动慢，结尾快（推荐用于退出）
- ease-in-out = 完整动效
- cubic-bezier 替代默认 linear

### 动效时长
- 微反馈：100-200ms
- 过渡：200-400ms
- 复杂动效：400-800ms
- 超过 1s：进入 storytelling 模式

## 第八篇：数据可视化

### Tufte 核心
- **Data-ink ratio**：越高越好
- **Chartjunk**：删 3D 阴影装饰图例网格
- **Small multiples**：同尺度复数
- "A table is nearly always better than a dumb pie chart"

### 视觉编码（pre-attentive）
- **Length 长度**：最有效（bar > pie）
- Position 位置：第二
- Angle 角度：中
- Area 面积：弱
- Color hue：弱（分类）
- Color saturation：弱（量化）
- Shape 形状：最弱（仅分类）

### 8 种定量信息
1. Time-series → line
2. Ranking → bar
3. Part-to-whole → pie/bar
4. Correlation → scatter
5. Frequency → histogram
6. Geographic → cartogram
7. Deviation → bar
8. Distribution → box plot

### Z 模式布局
dashboard 关键元素左上

### 误导图（避免）
- 截断 Y 轴（从 0 起）
- 3D 折线/柱
- 不当缩放
- 删除关键点
- 复杂图需解释 = 失败

## 第十三篇：东西方美学谱系

### 东方
- **wabi-sabi**：无常 / 不完美 / 未完成
- **shibui**：简朴 / 含蓄 / 谦逊 / 自然 / 日常 / 不完美 / 静默
- **Ma**：留白 = 关系
- **Kintsugi**：使用痕迹可见
- **Muji**：手绘原型
- **三宅一生**：服装即雕塑
- **Hygge**：温暖与极简并存
- **Japandi**：东方空 + 北方暖

### 西方
- **Bauhaus**：form follows function
- **De Stijl**：红黄蓝 + 黑白灰、矩形
- **Swiss Style**：网格 + 左对齐 + 不对称 + 客观
- **Mies**：Less is more
- **Rams**：Weniger, aber besser（10 诫）
- **Braun**：no-nonsense modernist
- **Vignelli**：Design is one
- **Eames**：Learning by doing
- **Pentagram**：horizontal organization

### 东方 ↔ 西方 共同点
- 都强调**少**
- 都强调**真实**
- 都强调**功能**
- 都强调**克制的表达**
- 唯一区别：东方更接受"不完美"，西方更追求"完美"

## 第十四篇：综合可执行设计配方

### 网页
- 基础栈：白/近黑 + 1 主色 + 12 阶色板 + 2 字体
- Hero：左对齐 64-80px display + 价值主张 + 1 CTA + 真实产品截图
- section 间距 96-128px
- Z 模式 / F 模式布局
- 参考：Linear / Stripe / Notion / Apple

### PPT
- 纯白或近黑 + 1 主色 + 灰阶 5 档
- 每页一个 idea，不堆 bullet
- 大数字 + 简短解释
- 左对齐为主
- 单色线性图标 (Lucide)
- 图表删 3D / 阴影 / 网格线 / 图例

### App
- 白 + 系统色板 或自定义
- 触控目标 44pt+
- 主操作 FAB / 顶部 bar
- 空状态自设计
- 不用 5 tab bar

### 海报
- 12 列 / Villard 网格先
- 真实摄影 > AI 美图
- Display 衬线 + 小号无衬线副文
- 大量纯色块 + 极少装饰

### 品牌
- Pentagram 模式：设计师 owned
- 跨媒介一致（Vignelli）
- 简洁标志（Rand）
- 字体即形象（Lubalin / Scher）

### 数据可视化
- 删 chartjunk
- 从 0 起的 Y 轴
- 用 length 比较（bar > pie）
- Small multiples
- Z 模式 dashboard
- 永远色盲友好

## 第十五篇：6 条核心心法

1. **先限制再创造**：1 主色 + 1 字体 + 1 圆角值
2. **做减法的勇气**：80% "加一笔"是错觉
3. **有一个主见**：决定气质（安静/张扬），通篇贯彻
4. **模仿具体对象**：照搬某个真实案例的色板+字体+圆角
5. **网格先于元素**
6. **真实 > 完美**：手绘 > AI 干净；轻微不对齐 > 完全居中

## 第十六篇：可复用 Prompt 模板

### 给任何 AI 设计任务：

```
设计要求（务必遵守）：

1. 配色：[主色 HEX] + 8 阶灰色（#F...→#1...，选带温度的：mauve/slate/sand）
2. 字体：[正文 family] + [标题 family]，最多 2 个
3. 字号：12/14/16/20/24/32/48/64，最多用 4 个 size
4. 对齐：所有内容左对齐，禁止居中（标题除外）
5. 间距：8 的倍数，section 间距 ≥64px
6. 留白：宁多勿少
7. 圆角：卡片 8px / 按钮 4px / 头像 999px（统一）
8. 禁用：渐变文字 / emoji 装饰 / 玻璃拟态 / 紫粉渐变 / 3D 元素 / AI 人脸
9. 主元素 1 个，次要用减弱的 color/weight
10. 气质：[极简/克制/编辑感/科技冷峻/温暖手作/日式禅意]
11. 视觉参考：[具体网站/品牌名 1-2 个]
12. 接受不完美：可材质纹理、轻微不对齐、留白呼吸
13. WCAG AA：色彩对比 4.5:1，不用颜色单独传达
14. 数据图表：删 3D，Y 轴从 0 起，长度比较
```
